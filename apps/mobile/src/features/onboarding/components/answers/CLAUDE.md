# onboarding/components/answers

## Purpose
One component per answer type used in the onboarding card flow.

## Contents
| File | Purpose |
|---|---|
| `YesNoAnswer.tsx` | Large Yes / No pill buttons |
| `SingleSelectAnswer.tsx` | Option chip list — one selectable |
| `MultiSelectAnswer.tsx` | Option chip list — multiple selectable |
| `TimePickerAnswer.tsx` | Native time picker (iOS wheel / Android clock) |
| `TextAnswer.tsx` | Free-text input with auto-focus |
| `MedicineFormAnswer.tsx` | Multi-field medicine entry (name, timings, food, criticality) |
| `CalendarConnectAnswer.tsx` | Google / Microsoft OAuth buttons with connected state |
| `PhotoUploadAnswer.tsx` | Image picker with circular preview |
| `CommuteDurationAnswer.tsx` | Fetches travel estimate then shows confirm/edit result |
